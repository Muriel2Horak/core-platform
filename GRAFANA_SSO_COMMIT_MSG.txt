- Iframe showed login page (302 redirect) instead of dashboard
- Dashboard returned 404 (user in wrong org)
- Missing provisioning logs and diagnostic tools

SOLUTION:

1. Backend (AuthRequestController):
   - Enhanced ensureUser/ensureOrgMembership/ensureActiveOrg with Step 1/3, 2/3, 3/3 logs
   - Fixed API calls: lookupUser() returns Optional<UserLookupResponse>
   - Added START/SUCCESS provisioning logs with emojis
   - Added imports: CreateUserResponse, UserLookupResponse, Optional

2. Frontend (grafanaUrl.ts):
   - Removed url.searchParams.set('orgId', String(orgId))
   - Backend setUserActiveOrg handles org selection (no URL param needed)
   - orgId parameter kept for API compatibility but unused

3. Grafana (grafana.ini):
   - Fixed jwk_set_url: http://backend:8080/.well-known/jwks.json (internal Docker network)
   - Verified JWT config: enabled=true, header_name=X-Org-JWT, org_id_claim=orgId, auto_sign_up=true
   - Verified security: allow_embedding=true, disable_login_form=true

4. Nginx (nginx-ssl.conf.template):
   - Verified /_auth/grafana has proxy_set_header Cookie $http_cookie (line 92)
   - Verified /core-admin/monitoring/ has proxy_set_header X-Org-JWT (line 119)
   - No changes needed - config is correct

5. Diagnostic CLI:
   - Created scripts/grafana/user-check.sh <login> (lookup user, list orgs, show current org)
   - Created scripts/grafana/user-set-org.sh <login> <orgId> (manually set active org)
   - Both use Grafana Admin API with basic auth

6. CI Integration:
   - Created scripts/grafana/sanity-test.sh (tests test_admin org 2 membership, current org, dashboard 200)
   - Added Makefile target: make test-grafana-sso
   - Added to help-advanced menu
